# main 分支保護設定（管理員）

本文件供 **Ohmomomo123** 在 GitHub 網頁操作，確保協作者**只能透過 Pull Request** 合併進 `main`，並由你審核。

## 設定步驟

1. 開啟：<https://github.com/Ohmomomo123/skin/settings/branches>

2. 點 **Add branch protection rule**

3. 填寫：

   | 欄位 | 建議值 |
   |------|--------|
   | Branch name pattern | `main` |

4. 勾選（建議全開）：

   - [x] **Require a pull request before merging**
     - [x] Require approvals: **1**
     - （可選）Require review from Code Owners
   - [x] **Do not allow bypassing the above settings**（連管理員也走 PR，較嚴謹；若你要能直接 push main 可不要勾）
   - [x] **Restrict who can push to matching branches** → 只選 **Ohmomomo123**（可選，最嚴格）

5. 按 **Create** 或 **Save changes**

## 設定完成後的行為

| 角色 | 可以做什麼 |
|------|------------|
| 協作者 | push 到 `feature/xxx`、`fix/xxx`；開 PR 請你審核 |
| 協作者 | **無法** 直接 push 到 `main` |
| 你（管理員） | 在 PR 頁 **Approve** → **Merge**；或依規則直接 push `main` |

## 審核 PR 檢查清單

- [ ] 變更內容與 PR 說明一致
- [ ] 沒有誤上傳 `.env`、模型檔、快取
- [ ] 若改 `app.py` / 前端，建議對方說明如何測試
- [ ] 合併後提醒對方 `git pull` 更新本機 `main`

協作者操作說明見 [協作指南.md](協作指南.md)。
